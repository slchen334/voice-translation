🚨 網頁上傳的人特別注意：用「拖檔上傳」時，.gitignore 不會幫你擋 .env！（.gitignore 只在用 Git 指令時才生效。）所以拖檔前，手動把 .env 這個檔挑掉、不要拖進去。
🔑 為什麼這麼嚴重：.env 裡是你的 Gemini／Tavily 金鑰。傳上 GitHub 等於公開給全世界，可能被拿去亂用、把你的額度或帳單燒爆。金鑰正確的家是 Render 儀表板（下一步就教）。
✅ 上傳後自我檢查：到 repo 的檔案列表看看有沒有 .env。有的話 → 點進去 → 右上角垃圾桶 → Delete 刪掉。（.env.example 是空白範例、沒有金鑰，留著沒關係。）
⌨️ 會用 Git 指令的人：git init → git add . → git commit → git push 即可，此時 .gitignore 會自動排除 .env。不熟就用上面的網頁上傳法，或把這幾行貼給 AI 請它教你逐步操作。② Render：建立服務

② 設定環境變數（關鍵！）
金鑰放這裡，不進程式碼
往下找到 Environment Variables，把金鑰一組一組加進去：

Key	Value
GEMINI_API_KEY	你的 Gemini 金鑰
TAVILY_API_KEY	你的 Tavily 金鑰（選填）
OPENAI_API_KEY	你的 OpenAI／Groq／DeepSeek 金鑰（選填；填了整站訪客免自填就能用 OpenAI 相容服務）
🔐 這就是為什麼程式用 os.getenv()——它會來這裡讀，金鑰不出現在程式碼或 GitHub。
💡 加了 OPENAI_API_KEY 這步驟跟 Gemini 完全一樣：New → Key/Value → Save。填了它，App 選「OpenAI 相容」時金鑰欄可留空（伺服器共用）。⚠️ OpenAI 是付費 API，記得去 OpenAI 帳戶設 Usage limit。② 部署！

Build 失敗？

多半是 requirements.txt 少套件或打錯字，看 log 補上。

啟動就掛？

確認 Start Command、app.py 用環境變數 PORT。

功能沒反應？

檢查環境變數金鑰有沒有填、有沒有拼錯。

第一次好慢？

免費方案喚醒中，等 30–60 秒。
